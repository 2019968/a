# 淘宝用户行为数据分析
基于淘宝公开用户行为数据集，使用 **MySQL + Excel** 完成从数据清洗、指标计算到可视化报告的全流程电商数据分析，用于巩固 SQL 实操能力、熟悉电商业务分析逻辑，为数据分析实习与求职积累实战项目。

## 技术工具
- MySQL：数据预处理、多维度指标计算、漏斗与 RFM 模型实现
- Excel：数据可视化、图表制作、分析结果汇总

## 分析流程
1. 数据预览与清洗，去除重复及异常数据
2. 用户行为分布、时段热度、类目偏好统计
3. 构建浏览—收藏—加购—购买转化漏斗
4. 基于 RFM 模型实现用户价值分层
5. 导出结果并完成可视化与业务结论整理

## 遇到的问题与解决
### 1. 转化漏斗口径错误导致转化率异常
初次计算漏斗时，因对行为链路逻辑理解不清晰，统计口径不统一，出现转化率异常偏高的问题。
重新梳理用户行为递进关系，明确各环节统计范围，避免同一行为重复计入，修正口径后得到符合真实业务的转化结果，深刻意识到数据分析中口径严谨性的重要性。

### 2. RFM 模型仅停留在代码复用，理解不深入
初期直接参考代码实现 RFM 分层，对 R/F/M 指标含义、分箱规则和业务价值理解较浅。
随后重新梳理模型逻辑，结合数据集实际分布确定分层阈值，对结果进行校验对比，确保用户分层合理可解释，完整掌握了 RFM 从理论到落地的实现思路。

## 学习收获
1. 熟练使用 SQL 完成数据清洗、分组聚合、子查询、时间处理等实战操作
2. 系统掌握电商核心分析思路：用户行为、转化漏斗、用户价值分层
3. 建立规范的数据口径意识，优先理清业务逻辑再进行代码实现
4. 能够独立完成从原始数据到可视化报表的完整分析流程
5. 提升数据解读与业务总结能力，学会用数据支撑简单运营建议
