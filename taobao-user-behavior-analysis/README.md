# 淘宝用户行为数据分析
基于阿里云天池公开数据集，使用 MySQL + Excel 完成的全流程电商用户行为分析。涵盖用户行为分布、时段/星期规律、转化路径诊断、RFM 用户分层与品类偏好，输出从清洗、计算、可视化到业务建议的完整实战闭环。

## 项目简介
- 数据集：淘宝用户行为数据集，约 514 万条行为记录
- 行为类型：浏览（pv）、收藏（fav）、加购（cart）、购买（buy）
- 分析目标：洞察流量结构与用户习惯，定位转化薄弱环节，建立用户价值分层，为精细化运营提供决策依据
- 技术栈：MySQL（数据预处理与指标计算）、Excel（数据透视表、图表、分析报告）

## 分析流程与核心发现
### 1. 用户行为整体分布
| 行为类型 | 次数      | 占比    |
|----------|-----------|---------|
| 浏览     | 5,145,668 | 94.24%  |
| 收藏     | 107,514   | 1.97%   |
| 加购     | 152,987   | 2.80%   |
| 购买     | 53,906    | 0.99%   |

从浏览到购买的转化路径长，整体购买行为仅占 0.99%，提升转化空间巨大。

### 2. 时段活跃度（小时维度）
- 流量最高峰：21 点（浏览 459,150，购买 3,960），22 点紧随其后
- 次高峰：20 点、19 点，晚间 19-23 点是核心购物时段
- 最低谷：凌晨 4 点（浏览 33,509），此时可基本不做资源投入
- 趋势总结：用户活跃度从早 6 点开始攀升，至晚间达到顶峰，符合典型电商“下班经济”特征

### 3. 星期活跃度
| 星期 | 浏览量 | 购买量 |
|------|--------|--------|
| 周四 | 832,966 | 7,830  |
| 周三 | 813,579 | 8,029  |
| 周二 | 813,045 | 8,016  |
| 周五 | 722,309 | 11,243 |
| 周一 | 654,559 | 6,478  |
| 周日 | 667,984 | 6,147  |
| 周六 | 641,226 | 6,163  |

周五购买量一枝独秀，是转化效果最好的一天；周四、周三浏览高但购买量相对平稳。周末流量不升反降，说明该用户群更偏向有目的的集中购买，而非周末闲逛型消费。

### 4. 多维度转化诊断
从“有过该行为的用户最终购买的比例”看不同行为的转化价值：

| 路径         | 上游行为用户数 | 购买用户数 | 转化率 |
|--------------|----------------|------------|--------|
| 浏览 → 购买  | 9,993          | 8,079      | 80.85% |
| 收藏 → 购买  | 5,679          | 4,952      | 87.20% |
| 加购 → 购买  | 7,962          | 6,995      | 87.85% |

收藏和加购用户的购买转化率均超过 85%，远高于整体浏览转化率，说明有明确兴趣/意向的用户价值极高。应重点优化收藏夹和购物车的二次触达，例如通过提醒、优惠券等工具进一步推高成交率。

### 5. RFM 用户价值分层
| 用户层级   | 人数   | 占比    |
|------------|--------|---------|
| 流失用户   | 2,670  | 33.05%  |
| 高价值用户 | 4,145  | 51.31%  |
| 挽留用户   | 1,264  | 15.65%  |

- 高价值用户（51%）：基本盘，需做 VIP 维护、个性化推荐、优先客服
- 流失用户（33%）：已流失或即将流失，需制定召回方案（大力度优惠券、短信/推送唤醒）
- 挽留用户（15.65%）：处在流失边缘，可通过签到、限时折扣等机制提高粘性

### 6. 商品类目偏好
| 类目 ID | 浏览次数 | 购买次数 | 转化率 |
|---------|----------|----------|--------|
| 10121   | 1,424    | 118      | 8.29%  |
| 4560    | 938      | 72       | 7.68%  |
| 2949    | 646      | 167      | 25.85% |
| 1911    | 358      | 34       | 9.50%  |
| 4751    | 213      | 20       | 9.39%  |

- 高流量低转化：类目 10121 浏览次数最高但转化率仅 8.29%，可优化详情页或调整选品
- 隐藏爆款：类目 2949 以 25.85% 的转化率领先，但曝光量仅为 646，应增加推荐流量
- 无加购直接购买：类目 1911、10121、12514 等无加购记录却有购买，可能是冲动消费品，适合在首屏展示

## 如何复现
1. 将淘宝用户行为数据集导入 MySQL（库名 `taobao`，表名 `user_behavior`）
2. 按顺序运行 `SQL/` 文件夹下的脚本：`data_preview.sql` → `data_cleaning.sql` → 各分析脚本
3. 导出查询结果至 Excel，用数据透视表或图表功能完成可视化
4. 最终报告详见 `Excel/taobao_user_analysis_report.xlsx`

## 遇到的坑与解决
- 漏斗口径错误：最初计算转化率时未按独立用户去重，导致转化率异常 > 100%。改用 `COUNT(DISTINCT user_id)` 后恢复正常
- 性能优化：原始数据量达 514 万行，计算 RFM 时通过建立组合索引，查询时间大幅缩短

## 业务优化建议
1. 强化购物车催付：加购用户购买转化率高达 87.85%，可在购物车页面设置“库存紧张”“限时优惠”提醒，并配置 1 小时未购自动推送
2. 集中火力在周五与晚间：周五购买爆发，20-22 点是黄金时段，适合安排秒杀、直播等重磅活动
3. 流失用户召回：33% 的流失用户仍有挽回价值，可设计“老客专属回归礼包”
4. 高价值用户优先服务：过半用户属于高价值层，应提供更完善的售后与会员权益，防止流失
5. 品类优化：降低高流量低转化类目（如 10121）的跳失率，将流量向高转化类目（如 2949）倾斜；对无加购直接购买的类目可设“快速购买”入口
